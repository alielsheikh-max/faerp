Pricing Control System

FAERP Overview & Guide

FAERP is a secure, high-performance web platform designed to streamline monthly supplier price collections, enforce strict profit margin controls with multi-layer managerial approval, and provide real-time pricing intelligence for sales agents.

Capabilities

2. Core Features Offered

Structured role-based modules across five user roles for clean separation of duties.

📦 A. Warehouse & Purchasing Module (Data Collection)

  • Guided Step-by-Step Collection: A clean, two-step wizard where users select a product category, then select specific items to record pricing. Supports multi-supplier row entry with inline transport fields and recommended supplier badges (⭐).
  • Negotiated Price Recording NEW: Purchasing officers can record negotiated prices alongside raw supplier quotes, with optional notes. The pricing engine gives negotiated prices priority over raw quotes when calculating selling prices.
  • Smart Duplication Warning (Change Requests): If a purchasing officer attempts to alter a price that has already been submitted and confirmed for the month, the app forces them to submit a Price Change Request with a mandatory business justification. Now also supports transport fee revisions.
  • Contextual Historical Matrix: While entering prices, the input screen displays a 3-month, 6-month, or full history comparison grid for each supplier so purchasing agents can immediately spot vendor inflation or anomalies.
  • Missing Quotes Tracker NEW: A dedicated panel showing all item×supplier combinations that haven't been quoted this month. Includes a quick-add modal to submit quotes directly from the missing list with inline transport fields and multi-supplier support.
  • Collection Overview Dashboard: Visual progress bars per category showing quote completion percentages, giving WH users an at-a-glance view of the month's collection status.
  • Rejected Quote Notifications: WH users receive notifications when their submitted quotes are rejected by SC, with a "Mark as Read" feature to manage their inbox.
  • Purchasing & Collection Reports: Generate real-time reports such as the **Price Collection Log** (itemized supplier quotes) and the **WH Monthly Summary** (supplier coverage and missing quote status).

📊 B. Pricing Control Module (SC — Controls & Analytics)

  • Real-time Pricing Engine: Allows SC managers to dynamically calculate selling prices based on the cheapest, average, or highest supplier quote for the month. The engine is tier-aware, automatically factoring in volume-based pricing tiers. Supports negotiated price priority — if a negotiated price exists, it takes precedence over raw quotes.
  • Flexible Margin Modes: Supports setting margins in three modes:
    • Percentage (%): Markup as a percentage of cost.
    • Fixed Amount (EGP): Absolute markup amount.
    • Divisor Mode NEW: Compute selling price by dividing cost by a factor (e.g., ÷ 0.85 = 17.6% margin).
  • Category-Level Bulk Pricing NEW: A dedicated /pricing/category page where SC can apply uniform markup to all items in a category at once using Percent, Fixed, or Divisor modes. Includes previous-month comparison and confirmed supplier auto-detection.
  • Volume-Tiered Pricing (Pallet System) NEW: SC Managers can define up to 4 pricing tiers per item based on quantity breakpoints (e.g., "1–10 units", "11–50 units", "51+ units / Pallet"). Each tier carries its own discount divisor, enabling granular bulk and pallet discount strategies. Tier pricing can be enabled/disabled monthly via the Admin policy panel.
  • Granular Margin Floor Enforcement: Enforces a strict profit floor at two levels:
    • Category-Level Floor: A default minimum margin applied to all items within a category.
    • Item-Level Floor (Override): A per-item floor that overrides the category floor when configured, allowing precision control for high-value or strategic products.
    • Both levels support Percentage (%) and Fixed Amount (EGP) floor types. The UI dynamically indicates which floor level is active and blocks publishing if the proposed price falls below the minimum acceptable markup.
  • Confirmed Supplier Selection: When publishing selling prices, SC can designate a confirmed supplier for each item. Items can also have a recommended supplier (⭐ badge) configured via Admin, which the pricing engine suggests by default.
  • SC Transport Override NEW: When enabled via monthly policy, SC can override the transport fee per item/month in the pricing engine, allowing fine-tuned logistics cost control.
  • Internal & SA Notes NEW: SC can attach internal-only notes (visible only to SC) and SA notification notes (pushed to Sales Agents when a price is published). Both are tracked in selling price history.
  • Live USD Currency Conversion NEW: Real-time EGP↔USD conversion powered by a live exchange rate API. The pricing toolbar shows a compact USD rate chip, and SC can export branded PDF/XLSX reports with USD pricing columns.
  • SC Overview Panel NEW: Circular progress indicators for pricing coverage, category-level completion tracking, supplier activity leaderboard, and a live recent-price-changes feed with percentage-diff indicators.
  • Independent Market Intelligence: High-end charts and analytical gauges on a dedicated /manager/analytics page:
    • Supplier Scorecard: Identifies cost leaders and vendor participation/coverage rates.
    • Month-over-Month Breakdowns: Visualizes volatility spread (Min, Max, Avg) per month.
    • Markup Strategy Advisor: Suggests markup strategies based on calculated volatility levels.
    • Price Trend Charts: SVG-based interactive price history visualization per item.
  • Consolidated SC Reports: Export dynamic reports such as **Published Selling Prices** (with custom month range selectors), **Monthly Market Overview** (market trend and min/avg/max spreads), **Supplier Price Comparison** (highlighting the cheapest rates), and **Price Volatility Alerts** (warning of multi-revision price changes).
  • Approval Dashboard: An inbox containing all price changes submitted by the warehouse. The SC Manager can approve or reject with a custom review note. Features tabbed views for Incoming Quotes and Pending Revisions.
  • Inline Monthly Review: A collapsible, side-by-side dashboard modal showing every item in the catalog. Managers can quickly inspect all supplier quotes (including negotiated prices) and submit selling prices item-by-item without leaving the screen.
  • Reconsidered Items Notifications: When the MG Manager returns items for reconsideration, SC receives them in the Notifications Center with the manager's reconsideration note, enabling a feedback loop between SC and MG.

👔 C. Manager Approval Module (MG — Oversight & Final Approval) NEW ROLE

  • Pricing Approvals Workstation: A full-featured dashboard where the Manager reviews all selling prices proposed by the SC team. The workstation displays status pills (Approved / Pending / Reconsidered / No Quote) with live counts, category filtering, search, and status-based filtering.
  • Three-State Approval Workflow:
    • Approve Single Item: Approves one specific item's selling price, making it visible in the Sales Catalog.
    • Approve All (Category): Bulk-approves all pending items within a category with one click.
    • Reconsider: Returns the item to SC with a mandatory reconsideration note, explaining why the proposed price needs revision. The item reverts to "reconsidered" status.
  • SA Price Hold: While a selling price is pending MG approval or has been returned for reconsideration, Sales Agents see the last-approved prices with a visible "⏳ Pending Revision 🔒" badge. Quoting on held items uses the previous approved range, ensuring business continuity.
  • Buying & Selling History: Each item row is expandable to reveal historical buying and selling price trends, giving the Manager full context before approving.
  • MG Reports: The Manager has access to published selling price reports and sales catalog exports for oversight purposes.
  • Notifications Center: Receives alerts when SC submits new pricing proposals for review, with pending count badges in the navigation.

🏷 D. Sales Agent Module (SA — Quoting & Deal Simulation)

  • Secure Price Catalog: Agents can view MG-approved min/max selling prices for the month. To protect purchasing leverage, raw buy costs are completely hidden from sales agents. When tier pricing is enabled, the catalog also displays tier-based price breakdowns.
  • Tier-Aware Deal Simulator: Agents select any approved item, type a target client quantity and unit price, and see the total contract value immediately. The simulator automatically detects and applies the correct pricing tier based on the entered quantity (Standard, Bulk, or Pallet), dynamically adjusting the allowed price range. Warns the agent if the price falls below the tier's minimum or above its maximum.
  • Visual Compliance Indicators: Displays green (compliant), yellow (above maximum), and red (below minimum allowed margin) warnings in real-time as agents type their pricing. Indicators reflect tier-specific price boundaries.
  • Pending Revision Handling NEW: Items awaiting MG approval display a "⏳ Pending Revision 🔒" badge. Agents can still view and quote using the last-approved price range, ensuring uninterrupted business operations while prices are under review.
  • Price Update Alerts & Acknowledgments NEW: Sales Agents receive banner alerts when selling prices are revised. They can acknowledge price changes, and the SC team is notified of acknowledgment status — creating a two-way confirmation loop.
  • Active Session Deal Board: Allows agents to combine multiple item quotes into a single deal board, check total profitability, and copy a formatted text summary to their clipboard for instant WhatsApp/email delivery to clients.
  • Sales Reports: Generate client-ready reports such as the **Approved Selling Price List** and the **Sales Price Catalog** (displaying dynamic stats cards for active periods, categories count, and catalog status).

⚙️ E. System Administration Module (AD — Master Data & Policy) NEW ROLE

  • Full CRUD Admin Panel: Manage items (with MOQ, transport per unit, recommended supplier), categories, suppliers (with extended contact fields: job title, email, region, address, represented products, fame name), and users (create/edit/delete with role assignment) from a dedicated admin interface.
  • Monthly Pricing Policy Configuration NEW: Toggleable monthly settings:
    • Tier Pricing Toggle: Enable or disable volume-tiered pricing system-wide for the active month.
    • SC Transport Override Toggle: Allow or restrict SC from overriding transport fees per item.
  • CSV Import System NEW: Import items and suppliers via CSV with drag-and-drop support, client-side CSV parsing with BOM handling, 8-row file preview before import, progress bars, and downloadable CSV templates.
  • Margin Floors Configuration: Dedicated interfaces for configuring margin floors per category and per individual item, including the override hierarchy (item-level overrides category-level).
  • Bulk Operations: Activate/deactivate items, move items between categories, and bulk delete with confirmation dialogs.
  • Activity Log Viewer NEW: A full browsable audit log at /admin/activity with pagination (60 events/page), filterable by role (AD/SC/WH/SA/MG) and event type. Tracks 20+ event types including sign-in/out, quote submissions, price changes, CRUD operations, and purge actions. Timestamped and color-coded.
  • Supplier Directory & Detail Pages NEW: Full supplier listing with search, region filter, and quote status filter. Click any supplier to open a rich detail modal with contact info, SVG line chart showing price history per item, quote history, and confirmed pricing tabs.
  • Item Detail Pages NEW: Individual item pages showing supplier history chart, pricing grid, and recommended supplier configuration.
  • About Page: System capabilities grid (8 modules), tech stack display (Next.js 14, React Server Components, TypeScript, SQLite), version info, and edition details.
  • Database Purge: Password-protected purge in the "Dangerous Zone" to clean the system for a fresh pricing cycle while retaining user records.

🌐 F. Core System Utilities

  • Multilingual RTL Support: Fully localized in English and Arabic. RTL layout adjustments (aligned grids, padding, and font sizes) are applied automatically on language toggle.
  • Theme Customizer: Toggle between clean light mode and premium dark mode interfaces.
  • Universal Smart Search NEW: Global search across items and suppliers with fuzzy matching on name, code, contact, region, and products. Opens rich inline detail cards showing recommended supplier badges, price history grids, and selling price history. Accessible via keyboard shortcut.
  • Role-Based Notifications Center NEW: A unified, role-specific notifications hub:
    • WH: Rejected quote notifications with "Mark as Read".
    • SC: Price acknowledgments from SA, pending change requests, negotiated price alerts, reconsidered items from MG.
    • MG: Pending pricing submissions from SC awaiting approval.
    • SA: Price update alerts with acknowledgment buttons.
  • Zero-Dependency PDF & Excel Generation:
    • Export dynamic price requests for vendors with zebra-striped, auto-fitting columns.
    • Export manager spreadsheets using `xlsx-js-style` with custom emerald headers, formatted currencies, and highlighted input fields.
    • USD pricing columns in exports when currency conversion is enabled.
    • On-screen print layouts optimized for physical archiving.
  • Automated Audit History: Collapsible visual timeline showing every past price modification, who made it, and the stated reason. Enhanced with a dedicated activity log viewer tracking 20+ event types across all roles.

4. VS Code & PDF Printing Instructions

How to View the Flowchart properly in VS Code

By default, VS Code's built-in Markdown preview window does not render Mermaid diagram syntax. Follow these steps:

  1. Open VS Code and navigate to the Extensions tab on the left sidebar (or press Ctrl+Shift+X).
  2. Search for "Markdown Preview Mermaid Support" (published by Matt Bierner) and click Install.
  3. Open CLIENT_OVERVIEW.md.
  4. Press Ctrl+K V (or open the Command Palette with Ctrl+Shift+P and search for "Markdown: Open Preview to the Side").
  5. The flowchart under Section 1 will render dynamically inside the preview tab on the side.

How to Convert this Document & Flowchart to PDF

We have engineered this standalone HTML file to render everything in vector clarity. To save as a PDF:

  1. Double-click CLIENT_OVERVIEW.html to open it in Chrome, Edge, Safari, or Firefox.
  2. Press Ctrl+P (Windows) or Cmd+P (macOS) to open the print dialog.
  3. Set the **Destination** to "Save as PDF".
  4. Set **Layout** to "Portrait".
  5. Under **More Settings**, tick the box for "Background graphics" (CRITICAL to ensure color backgrounds and flowchart blocks load correctly).
  6. Click **Save** and select your local output folder.
Installation Guide

5. Setup Guide for First-Time Use

📋 System Prerequisites

Ensure **Node.js (v18.0.0 or higher)** is installed on your local machine. NPM is included automatically with Node.js.

🚀 Step-by-Step Installation

  1. Unzip / Extract: Ensure all project folders (including src, package.json, etc.) are in your local workspace.
  2. Install Dependencies: Open a terminal inside the root directory and run: 
    npm install
  3. Launch App: Start the Next.js development server: 
    npm run dev

    *The application automatically creates the SQLite database at `data/faerp.sqlite` and seeds the tables with default products, categories, suppliers, and historical quotes on first boot.*

  4. Access the App: Open your web browser and navigate to: http://localhost:3000

🔑 Seeded Demo Credentials

Log in with the following default credentials to test each specific view:

🛡️ Administrative Database Purge Function

A password-protected database purge panel is built in to allow administrators to clean the system for a fresh pricing cycle. This deletes all catalog inventory, suppliers, quotes, price requests, and margins, while retaining user authentication records.

To execute a database wipe:

  1. Log in with the System Admin account (admin / admin123).
  2. Navigate to the Admin Panel (/dashboard/admin).
  3. Scroll to the bottom of the page to find the Dangerous Zone.
  4. Type the purge confirmation password: 17012911
  5. Click Wipe All Data Except Users. Double-confirm the browser prompt.
